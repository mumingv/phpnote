# 委托模式

## 基本概念

委托设计模式（Delegate Design Pattern）通过分配或委托至其他对象，能够去除核心对象中的判决和复杂的功能性。


## 使用场景

1. 基类对象实例化的时候，同时根据类型实例化委托类的对象，并将委托类对象保存在基类对象中；
2. 基类对象提供接口函数，接口函数中调用各委托类对象的相应函数；
3. 在扩展新的委托类的时候，无需修改基类的代码，仅新增新的委托类并实现对应的接口即可。
